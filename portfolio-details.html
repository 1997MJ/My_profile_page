현재 어떤 재난 키워드가 SNS에서 자주 언급되고 있는 지를 시각화해서 보여주는 웹 사이트

최근 코로나바이러스 감염증, 태풍 힌남노, 이태원 압사 사고, 튀르키예-시리아 지진 등 재난 상황이 빈번하게 발생하면서 뉴스, 소셜 미디어, 재난 문자, 재난 상황에 있는 사람에게 연락하는 방법 등으로 재난 상황을 알 수 있었다. 하지만 소셜 미디어를 포함해 한눈에 재난 상황을 볼 수 있는 매체는 없었다.
  그리하여 소셜 미디어의 실시간 데이터를 통해 재난 상황을 한눈에 보여주는 매체를 설계 및 구현하였다.
  트위터, 인스타그램, 네이버 블로그에서 데이터를 크롤링하여 데이터베이스에 저장하고 데이터베이스에서 유의미한 데이터만 사용하기 위해 KSS를 이용한 필터링 과정과 k평균 알고리즘을 이용한 학습 과정을 거친 동적 웹 페이지를 통해 정제된 데이터를 이용자에게 시각적으로 제공하는 웹사이트이다.

SNS 크롤링을 통한 실시간 웹 페이지 구성도

실시간 웹 페이지 동작 과정

동작하는 순서는 각 인스타그램, 트위터, 네이버 블로그 및 기사를 특정 시간과 설정한 키워드에 맞게 크롤링을 하면 크롤링한 데이터들을 유의미한 데이터로 분류하고 정제한다.
  이후 정제된 데이터들을 데이터베이스에 저장한 뒤 웹 페이지의 형식에 맞게 데이터를 시각화해 웹 페이지 이용자들에게 보여주는 식으로 동작한다.

SNS 데이터 크롤링

SNS 데이터를 수집 방법

재난과 관련된 SNS 데이터를 수집하는 방법은 아래와 같이 동작한다.

  • 재난 키워드를 설정한다. [지진, 태풍, 홍수, 코로나, 교통사고, 화재]
  • 데이터 중복을 방지하기 위해 현재 시간에서 30분 전에 시간을 가져온다.
  • 크롤링 기술을 사용하여 최신순으로 키워드를 검색한다.
  • 재난 키워드를 언급한 SNS 데이터를 가져온다.
  • 데이터에서 업로드 시간을 가져와서 순서 2에 저장한 시간과 비교한다.
  • 비교 후 시간이 초과 되면 설정한 다음 키워드로 키워드가 변경된다.
  • 모든 키워드를 검색하면 다음 SNS에 넘어가 순서 3부터 6까지의 과정을 반복한다.
  • 모든 SNS에서 데이터 수집이 끝나면 크롤링 함수가 종료된다.
  • 데이터 전처리 함수가 실행된다.
  • 30분 이후 위의 과정을 다시 반복한다.

SNS 데이터 전처리 및 필터링

SNS 데이터 전처리 및 필터링 과정

재난과 관련된 SNS 데이터를 수집하는 방법은 아래와 같이 동작한다.

  • 수집한 데이터를 MYSQL을 통해 전부 가져온다.
  • 조건문을 통해 데이터 수가 20개보다 많은 지 비교한다. 20 개보다 적으면 전처리 함수가 종료된다.
  • 가져온 콘텐츠 값의 이모티콘 및 이모지를 제거한다.
  • kss 모듈을 통해 콘텐츠 값을 문장 단위로 분리한다.
  • 분리된 모든 문장을 word2vec을 통해 모두 벡터화시킨다.
  • Kmeas 클러스터링을 통해 가져온 모든 문장을 군집화시킨다.
  • ['코로나 신규 확진자가 발생했다.', '화재가 발생했다.', '홍수 피해가 발생했다.', '사고 났다.', '눈이 많이 내린다.', '산에 불이 났다.', '붕괴 사고가 발생하다.', '폭발하다.', '태풍이 발생했다.']의 문장들이 속해져 있는 범주(인덱스) 값을 가져온다.
  • 반복문을 통해 가져온 문장들이 해당 범주에 속해 있는지 비교한다. 속해 있지 않으면 해당 데이터는 저장하지 않는다.
  • 반복문을 통해 초기에 설정한 욕설 필터링을 통해 욕설이 포함되어 있는지 확인하는 비교 과정을 거친다. 데이터에 욕설이 포함되어 있으면 해당 데이터를 삭제한다.
  • 최종적으로 전처리 및 필터링 된 데이터를 MYSQL을 통해 저장한다.
  • 해시태그의 데이터를 늘리기 위해서 kkma 모듈을 통해 필터링 과정을 거친 데이터 콘텐츠의 명사 값을 가져와 해시태그 테이블에 추가적으로 저장한다.
